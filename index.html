Industrial OMR Sheet Generator Documentation

Optical Mark Recognition (OMR) Architecture

This tool generates Level 3 Industrial OMR Sheets designed for hardware scanners (Sekonic, OpScan, Scantron) and software-based vision systems (ZipGrade, OMR APIs). Unlike standard HTML forms, this generator adheres to strict physical constraints required for optical reading:

  • Dropout Colors: The grid utilizes specific Hex codes (Pink/Red spectrum) that drop out under Infrared (IR) scanning, leaving only the carbon pencil marks visible to the sensor.
  • Timing Tracks: The vertical and horizontal black blocks (Timing Tracks) provide the scanner with a coordinate system to map the paper's velocity and skew.
  • Fiducials: Corner markers ensure image processing algorithms can rectify the perspective of the sheet if scanned via mobile cameras.

User Guide & Troubleshooting

How do I configure the columns?
The Grid Engine automatically distributes questions. If you select 100 Questions and 4 Columns, the system calculates 25 rows per column. If the layout looks too dense, increase the number of columns to 5 or reduce the question count.
Is this compatible with Python/OpenCV?
Yes. The sheet includes standardized Fiducials (Top-Left, Top-Right, Bottom-Left, Bottom-Right). A Python script using `cv2.findContours` can easily locate these anchors to perform a perspective transform before reading the bubble grid.
Printing Instructions
1. Use A4 Paper (100 GSM recommended).
2. Set Printer Scaling to "100%" or "Actual Size". Do NOT use "Fit to Page".
3. Laser printers provide better edge definition for the timing tracks than inkjet printers.
